mute.fm Release Notes



0.9.14 April 1, 2014
  • No longer expires, even on program start.


0.9.13 January 31, 2014
  • No longer expires
  • You can now make donations! :)


0.9.12 December 15, 2013
  • Expiration extended
  • Exit dialog should look better for bigger fonts
  • Fixed bug when getting nullpointer exception in music chooser
Note: If using AIMP3 and it doesn't detect, manually edit the sound info's program name.


0.9.11 November 14, 2013
  • Expiration extended
  • Be more consistent with mute.fm+ labels
  • Allow minimize when exiting from systray
  • Don't show warning dialog upon system shutdown
  • Treat "" like None for JS and DOS operations
  • Coming soon:
    • Properly ignore notifications (i.e. Facebook alerts)
    • Work with AIMP3 (bug reproducible, understood)
    • perhaps other fixes based on user feedback


0.9.10 October 1, 2013
  • mute.fm+ uses newer Awesomium (1.7.2)
  • Expiration extended


0.9.9 September 3, 2013
  • Growl no longer installed by default or included within installer.
  • Split into mute.fm and mute.fm+ (mute.fm without web support or Growl is now a 90% smaller download)
  • Added support for Zune and Windows Media Player.
  • Fixed bug where iTunes would hang when started second time.
  • Music after installing is now a Bach MIDI played in Windows Media Player instead of a YouTube video (needed for mute.fm without web support)
  • Active Sounds changed to Recent Sounds and you can change if they should cause music automuting
  • Changed behavior for when music is stopped to save bandwidth: defaults at 30 min (instead of 5 min) and should prompt user to show window.
  • Misc bugs fixed.


0.9.8 August 27, 2013
  • Better status messages in main UI
    • Always tells user what mute.fm is thinking and what actions are available
    • Countdown shown only when it should
    • Warning that web music not playing moved from browser to main ducker ui
    • Shows notification if no sound playing for several seconds to let the user know they may need to interact with the player
  • Spotify and WinAmp support
  • New system tray balloon notification option as a Growl alternative. Warning that user interaction required for player can be turned off.
  • Improved reliability and fixed edge cases for automatically killing music to save bandwidth and restoring music later.
  • Web-based music comes back while fading in more reliably.
  • Lots of bug fixes and minor changes.
  • Removed 'toggle foreground and background sounds' temporarily
  • Currently broken: Grooveshark (page won't load), Windows Media Player (inconsistent), Zune (more inconsistent)


0.9.7 August 12, 2013 -- Release pulled


0.9.6 August 6, 2013
  • Detects background music that was run prior to launching mute.fm
  • Added playpause support (doesn't try to play if paused). Foobar2000 should use playpause instead of pause so that music fades in where it was paused. If upgrading, reload defaults to set this.
  • Only runs flash/growl installers when needed. Eliminates error dialogs.
  • Snappier UI after playing sound
  • New options: set poll interval, can create hotkey for 'show'
  • mute.fm browser processes renamed to 'mute_fm_web' instead of 'awesomium'
  • Lots (but not complete) clean up of logging (and fixing some issues that had been logged)
  • Better support for zune (occasionally gets confused though); media player seems to be working well
  • Various bug fixes


0.9.5 July 29, 2013
  • New hotkeys window
  • New "Reload icons" button. Useful if program loaded without Internet.
  • Shows status messages in main UI (useful if not using Growl)
  • New export feature (for if you add custom commands to your background music)
  • Fixed another (smaller) memory leak
  • Misc bug fixes


0.9.4 July 25, 2013
  • User can view/change timeouts
  • Hotkey assigned to restart music immediately (currently Ctrl-Alt-X)
  • Hotkey assigned to toggle foreground and background sounds (useful during commercials); (currently Ctrl-Alt-Z)
  • UI now shows a countdown for how long to wait for music to return (or you can click Play or use the hotkey to start it immediately)
  • Lowered requirement to .NET 4 Client Profile
  • Fixed a memory leak (should use much less memory over time now)
  • Misc bug fixes
Noticed that processes that use multiple audio sessions (such as Windows Media Player and Zune) aren't supported properly yet.


0.9.3 July 19, 2013
  • Uses native controls for the main UI to improve load time
  • Has a simpler main UI by achieved by
    • moving music background selection into a dialog
    • removing volumes entirely (this is no longer a volume mixer replacement)
    • only showing foreground sounds that are playing now (instead of everything visible in Windows volume mixer)
  • Has a new settings UI allows adding/editing/removing background musics, including custom commands for pause, etc.
  • Uses Growl for notifications instead of writing on top of the screen; cleaner and gives user more control
  • Now has volume controls within the embedded browser
  • Lets the user click Play instead of waiting for music to fade back in
Note: The config file was restructured a bit so if you have problems, do Options->Reset Settings or delete [userdir]\appdata\roaming\mute.fm\config.json
